1. Introduction.

‘‘I never intended HTML source code (the stuff with the angle brackets) to be seen by users...’’

[BER 2001]

Bracy (pronounced bray-see) is a minimalist document compiler. It makes HTML [POW 2001] files that are supposed to look like printed pages when displayed by a web browser. (The page you’re reading now was made by Bracy.) Most browsers aren’t very good at rendering text, but Bracy tries as hard as it can anyway. Bracy also tries to produce HTML that’s nicely indented, so it’s readable by Humans (not just programs). This is useful if you want to edit an HTML file made by Bracy, but you don’t have Bracy, or you don’t have the Bracy source that produced the HTML file, or both.
      HTML’s syntax is incredibly verbose and inconsistent: some features are implemented as tags, others as attributes of tags, and still others as styles associated with tags, all without apparent design principles. And despite claims to the contrary, there is no standard HTML, since no two browsers seem to agree on what HTML’s constructs mean, or what subset of them should be implemented. As a result, HTML is nearly useless to the programmer who wants his or her code to work the same way on any computer. (Please don’t write to say that HTML was created by the gods and it’s a sin for me to criticize it, or any similar nonsense.)
      The important word in the previous paragraph was nearly. HTML does seem to have a few constructs that work almost the same way in all browsers. Bracy tries to use these few constructs exclusively. It hides them behind a simpler notation so you don’t have to see them, or even think about them. The good news, then, is that Bracy is easier to use than HTML. The bad news is that Bracy can’t do everything HTML can.
      Bracy is written in the experimental programming language Orson [MOE 2017], which I designed and implemented. One reason I wrote Bracy is to find bugs in my Orson compiler. I found lots of them. I also wanted a relatively painless way to write Orson documentation in HTML. There are mechanisms in Bracy to facilitate this, but you can still use Bracy even if you don’t care about Orson.

2. Running Bracy.

Bracy works under Unix-like operating systems, such as GNU/Linux. This section assumes that Bracy is already installed on your system.
      To run Bracy, you must first make a file containing Bracy source code, using your favorite text editor. (Bracy source code will be described in the remainder of this manual.) The name of the Bracy source file must end with the suffix ‘.by’.
      Bracy source files use 31-bit characters, encoded using UTF-8. This includes the familiar 21-bit Unicode characters [UNI 2006] and the 7-bit ASCII characters [ANS 1986] as subsets. If you write source files on a Unix-like system in Unicode or in ASCII, then you don’t need to care about this: everything should work fine.
      Suppose you have a Bracy source file called Manual.by. You can translate it to HTML by typing the command bracy Manual.by to the Unix shell. Bracy then makes an object file called Manual.html that contains HTML code. This file can be viewed with a web browser. If your source file has syntax errors, then Bracy will also print messages that tell you where the errors appear.
      Bracy makes HTML object files using ASCII characters exclusively. A character not in ASCII is rendered as ‘&#d;’ where d is a series of one or more digits, the character’s decimal numeric code. For example, Bracy renders the character α (Greek lower-case alpha) as ‘&#945;’. Some browsers can’t display characters like these.
      The bracy command has options that are not discussed here. Type man bracy to see what they are, and how to use them. You can also type man ascii to find out more about the ASCII character set. On many systems you can type man unicode and man utf-8 to find out about Bracy’s character encoding.

3. Syntax.

Bracy’s syntax was inspired by that of the document compiler SCRIBE [REI 1980]. The syntax has two parts. The first deals with styles, which describe how characters will appear. The second deals with scopes, most of which describe how text is rendered into HTML.

3.1. Styles.

Bracy has ten character styles. They were chosen partly for their potential usefulness, and partly because they are likely to be supported by all web browsers [POW 2001]. Each style is written as {style text}, where style is a series of one or more characters that tells what style you want, and text is the text that you want to appear in that style.

  • {b Abc} gives you Abc in boldface.

  • {c Abc} gives you ABC in small capitals.

  • {i Abc} gives you Abc in italics.

  • {g Abc} gives you Abc in gray.

  • {s Abc} gives you Abc struck through.

  • {t Abc} gives you Abc in typescript.

  • {u Abc} gives you Abc underlined.

  • {+ Abc} gives you Abc in superscript.

  • {- Abc} gives you Abc in subscript.

  • {- Abc {= Ez}} gives you Abc Ez in subscript and sub-subscript.

Bracy simulates the small capitals style by using upper case letters in a reduced size. Also, underlined words displayed with ‘u’ are not links, and must not be confused with them.
      Character styles can be nested, so that {b {i Abc}} gives you Abc in boldface italic. Similarly, {i {c {t Abc}}} gives you ABC in italic small capital typescript. In principle, you should be able to get any combination of styles, but in practice, some combinations may not look good with your browser. For example, using ‘+’ and ‘-’ together may give results different from what you expect.
      Bracy doesn’t care about the order of nested styles, so {i {b Abc}} gives you the same results as {b {i Abc}}. Duplication of styles has no effect, so if you write {b {b Abc}}, then you’ll get the same results as {b Abc}; you won’t get boldface that’s twice as bold. You can abbreviate a nested style like {b {i Abc}} by writing {bi Abc} or {ib Abc}, since order doesn’t matter. You could even write {bibi Abc}, since duplication doesn’t matter either, but that would be silly. You can abbreviate any number of nested styles in this way.

3.2. Braces, backslashes, and blanks.

As the previous examples suggest, Bracy gets its name because it uses the open brace ‘{’ and the closing brace ‘}’ as grouping symbols. Some people call these characters ‘‘curly brackets.’’
      If you want to show braces in your text, then you must prefix them with backslashes, as in ‘\{’ and ‘\}’. Braces prefixed in this way will be treated as ordinary characters, not as grouping symbols. If you want to show a backslash in your text, then you must also prefix it with a backslash, as in ‘\\’. For example, if you write ‘\{\\\}’, then the browser will display it as ‘{\}’.
      When a web browser is displaying your text, it might break a line where a blank appears. Sometimes you don’t want that to happen. You can avoid this problem by writing a blank prefixed by a backslash: ‘\ ’. This is called a nonbreaking blank. For example, if you write ‘a\ b\ c’ then Bracy will make sure a, b, and c all stay on the same line.

3.3. Scopes.

Bracy uses scopes to determine how your text should be organized into lines and paragraphs, and how those lines and paragraphs should be displayed. A scope is written as {scope text}, where scope is the name of the scope. Depending on which scope is being used, the text may be a series of words, a series of lines, a series of paragraphs, a series of scopes, or a series of both words and scopes. These are defined in the following way.

  • A newline is the invisible character that you get when you press the RETURN key or the ENTER key.

  • A word is a series of one or more characters, terminated by a blank, a newline, or a brace.

  • A line is a series of zero or more words, separated by one or more blanks, and terminated by a newline or a closing brace.

  • An empty line is a line with zero words: all it has is a newline or a closing brace at the end.

  • A paragraph is a series of one or more non-empty lines, terminated by an empty line or by a closing brace.

A Bracy source file is a series of zero or more scopes. These scopes are described in the rest of this manual.

3.3.1. Left.

The left scope is simplest. It displays a series of paragraphs whose lines are flush left and filled. The first paragraph is not indented, but the second and later paragraphs are. The text in most web pages appears this way. For example, if your source file contains this:

{left 
  {b Thelma Suzette Bracy} (January 2, 1919 – 
  October 9, 1967) was an American science fiction writer. 
  Along with Isaac Asimov, L. Sprague de Camp, Robert 
  Heinlein, and A. E. van Vogt, she was one of many new 
  writers discovered in the late 1930’s by John W. Campbell, 
  editor of {i Astounding Science Fiction} magazine. 
 
  Bracy is well known for her short stories that eerily 
  predicted many aspects of modern computer technology, 
  including cell phones, programming languages, and the 
  Internet. 
  Some claim that the name of the Internet language {c html} 
  was obtained by permuting the consonants of Bracy’s first 
  name, but there is no evidence for this. 
 
  In many of Bracy’s stories, the protagonist is Dr. Jules 
  Webster, a private detective with a doctorate in 
  ‘‘electrolinguistics.’’ 
  The ‘‘electronic brain’’ owned by a large organization 
  mysteriously fails, and the reclusive Webster is hired to 
  repair it. 
  He does so by typing a cleverly written ‘‘logigram’’ into 
  the brain, using a ‘‘keypunch.’’}

then the browser will display these three paragraphs:

Thelma Suzette Bracy (January 2, 1919 – October 9, 1967) was an American science fiction writer. Along with Isaac Asimov, L. Sprague de Camp, Robert Heinlein, and A. E. van Vogt, she was one of many new writers discovered in the late 1930’s by John W. Campbell, editor of Astounding Science Fiction magazine.
      Bracy is well known for her short stories that eerily predicted many aspects of modern computer technology, including cell phones, programming languages, and the Internet. Some claim that the name of the Internet language HTML was obtained by permuting the consonants of Bracy’s first name, but there is no evidence for this.
      In many of Bracy’s stories, the protagonist is Dr. Jules Webster, a private detective with a doctorate in ‘‘electrolinguistics.’’ The ‘‘electronic brain’’ owned by a large organization mysteriously fails, and the reclusive Webster is hired to repair it. He does so by typing a cleverly written ‘‘logigram’’ into the brain, using a ‘‘keypunch.’’

Biographical information about Thelma Bracy is from the introduction to Beyond Computation: The Complete Short Fiction of Thelma Bracy [CHA 2004].

3.3.2. Right.

The right scope displays a series of paragraphs whose lines are flush right and filled. Paragraphs are not indented. This usually works well only for short texts, such as quotations. For example, this:

{right 
  Computers don’t really work. 
  I just knew that before you did. 
 
  — Thelma Bracy (1966)}

will appear like this:

Computers don’t really work. I just knew that before you did.
— Thelma Bracy (1966)

3.3.3. Justify.

The justify scope displays a series of paragraphs whose lines are justified and filled. The first paragraph is not indented, but the second and later paragraphs are. Text in books and papers is typically justified, although text in most web pages is not (this page isn’t typical). For example, the source text:

{justify 
  Bracy’s first published story, ‘‘Fruit Counter,’’ 
  appeared in the March 1938 issue of {i Astounding.} 
  The electronic brain that runs the assembly line of the 
  National Canned Produce Company is inexplicably putting 
  pears into peach cans. 
  Baffled company engineers call Jules Webster in to 
  investigate. 
 
  When Webster arrives, he thinks for a few moments, 
  then asks to see a can of peas. 
  The engineers protest that the problem is with pears, 
  not peas, but he repeats his request. 
  When they get him a can of peas from the warehouse, he 
  opens it—and it contains pears, too! 
 
  A grinning Webster explains to the astonished engineers 
  that they have mistakenly instructed the brain to recognize 
  only the first three letters of each fruit and vegetable. 
  Since ‘‘pears,’’ ‘‘peaches,’’ and ‘‘peas’’ begin with the 
  same three letters, the brain cannot distinguish them. 
  In a dramatic keypunching session, Webster changes a 3 to 
  a 4, and the problem is solved.}

will be displayed like this:

Bracy’s first published story, ‘‘Fruit Counter,’’ appeared in the March 1938 issue of Astounding. The electronic brain that runs the assembly line of the National Canned Produce Company is inexplicably putting pears into peach cans. Baffled company engineers call Jules Webster in to investigate.
      When Webster arrives, he thinks for a few moments, then asks to see a can of peas. The engineers protest that the problem is with pears, not peas, but he repeats his request. When they get him a can of peas from the warehouse, he opens it—and it contains pears, too!
      A grinning Webster explains to the astonished engineers that they have mistakenly instructed the brain to recognize only the first three letters of each fruit and vegetable. Since ‘‘pears,’’ ‘‘peaches,’’ and ‘‘peas’’ begin with the same three letters, the brain cannot distinguish them. In a dramatic keypunching session, Webster changes a 3 to a 4, and the problem is solved.

3.3.4. Center.

The center scope displays a series of lines, each of which is centered. Paragraph breaks are rendered as empty lines. For example, this:

{center 
  {b Peas and Queues} 
 
  Thelma Bracy 
  February 13, 1938}

Produces this:

Peas and Queues

Thelma Bracy
February 13, 1938

As the example suggests, the center scope works best with short texts. It’s often used to make headings and titles.

3.3.5. Bullet.

The bullet scope displays a series of scopes and words with a symbol in front of each. The symbol is chosen by your browser, but it’s typically some kind of raised dot, called a ‘‘bullet.’’ For example, this:

{bullet 
  {left 
    Thelma Bracy was born in Far Rockaway, New York.} 
  {left 
    Her family was of French Canadian ancestry. 
    She had two older sisters, Annette and Yvette.} 
  {left 
    Bracy loved science, but her sisters teased her, 
    saying she couldn’t be a scientist. 
    She began reading science fiction magazines instead.} 
  {left 
    When she was 10, she wrote her first science fiction 
    story, in which portable telephones secretly track 
    their owners’ movements.}}

becomes this:

  • Thelma Bracy was born in Far Rockaway, New York.

  • Her family was of French Canadian ancestry. She had two older sisters, Annette and Yvette.

  • Bracy loved science, but her sisters teased her, saying she couldn’t be a scientist. She began reading science fiction magazines instead.

  • When she was 10, she wrote her first science fiction story, in which portable telephones secretly track their owners’ movements.

Even though left scopes were used in this example, any scopes may appear inside bullet scopes. I could have used justify scopes, or some could be left scopes and others could be justify scopes, etc.

3.3.6. Column.

The column scope lets you put many scopes in a place where only one would ordinarily be allowed. For example, this bullet scope has a column scope nested inside it.

{bullet 
  {left 
    Bracy attended City College in New York, where she 
    majored in mathematics.} 
  {left 
    While in college, she submitted stories to science 
    fiction magazines like the ones she had read as a 
    child.} 
  {column 
    {left 
      Her stories were about a mathematical detective named 
      Jules Webster, who lived thirty years in her future.} 
    {bullet 
      {left 
        She named him after Jules Verne, her favorite 
        author.} 
      {left 
        She called him Webster because she liked to read 
        dictionaries.}}}} 
  {left 
    All her stories were rejected.}}

It will be displayed like this. When you nest one bullet scope inside another, the browser may choose different bullet symbols for inner scopes than for outer scopes.

  • Bracy attended City College in New York, where she majored in mathematics.

  • While in college, she submitted stories to science fiction magazines like the ones she had read as a child.

  • Her stories were about a mathematical detective named Jules Webster, who lived thirty years in her future.

    • She named him after Jules Verne, her favorite author.

    • She called him Webster because she liked to read dictionaries.

  • All her stories were rejected.

The column scope must have at least one scope nested inside it, but it’s silly to use it with less than two nested scopes.

3.3.7. Number.

The number scope is like the bullet scope, but it displays a series of scopes and words with a number in front of each. The numbers start at one, and are incremented for each scope and word. For example, this:

{number 
  {left 
    One day Bracy got a letter from John W. Campbell at 
    {i Astounding} that wasn’t a rejection.} 
  {left 
    He suggested she write stories in which Jules Webster 
    solves mysteries involving ‘‘electronic brains.’’} 
  {left 
    She wrote a story called ‘‘Peas and Queues’’ and sent 
    it to Campbell.} 
  {left 
    He bought it, but changed the title to ‘‘Fruit Counter’’ 
    for publication. 
  {left 
    By the time she graduated from college, Bracy had sold 
    Campbell six more stories.}}

becomes this:

  1. One day Bracy got a letter from John W. Campbell at Astounding that wasn’t a rejection.

  2. He suggested she write stories in which Jules Webster solves mysteries involving ‘‘electronic brains.’’

  3. She wrote a story called ‘‘Peas and Queues’’ and sent it to Campbell.

  4. He bought it, but changed the title to ‘‘Fruit Counter’’ for publication.

  5. By the time she graduated from college, Bracy had sold Campbell six more stories.

The browser chooses the format in which the numbers will be displayed. If you nest one number scope inside another, the browser may show the numbers in the inner scopes differently from those in outer scopes.

3.3.8. Itemize.

The itemize scope resembles the bullet and number scopes. It displays a series of nested scopes and words with a short text (often a single word) in front of each. For example, this:

{itemize 
  {left Fruit Counter.} 
  {justify 
    Webster repairs an electronic brain that recognizes 
    only the first three letters of fruit and vegetable 
    names.} 
  {left Dangerously Unbalanced.} 
  {justify 
    Webster narrowly escapes death after the Army’s 
    electronic brain gets keypunched logigrams with 
    too many parentheses.} 
  Policy. 
  {justify 
    Webster is called to Washington D.C. when an 
    electronic brain forces everyone to buy insurance 
    they do not want.} 
  {left Give Me a Break.} 
  {justify 
    The world’s telephones stop working when someone 
    forgets to keypunch a special word into the 
    electronic brain that controls them.}}

becomes this:

Fruit Counter.

Webster repairs an electronic brain that recognizes only the first three letters of fruit and vegetable names.

Dangerously Unbalanced.

Webster narrowly escapes death after the Army’s electronic brain gets keypunched logigrams with too many parentheses.

Policy.

Webster is called to Washington D.C. when an electronic brain forces everyone to buy insurance they do not want.

Give Me a Break.

The world’s telephones stop working when someone forgets to keypunch a special word into the electronic brain that controls them.

The odd-numbered words or scopes inside itemize state things to be defined, and the even-numbered ones state their corresponding definitions. As a result, the itemize scope must have an even number of things nested inside it.
      Again, I could have used scopes other than justify in this example. I could have used bullet, left, number, another itemize, or a mixture of these, along with column. However, it’s best to keep things simple, since some browsers can’t display complex text in a visually appealing way.

3.3.9. Display.

The display scope is unlike the scopes you’ve seen so far, because it works with lines instead of words and paragraphs. It makes the browser show a series of unfilled lines, flush left, in which all blanks are treated as nonbreaking blanks. This is useful for poetry or other irregularly indented English text. For example, this limerick is due to an anonymous wag at the 1941 World Science Fiction Convention in Denver, Colorado.

{display 
There is a young woman called Bracy, 
     whose clothes aren’t frilly or lacy. 
She expertly writes 
     about man-machine fights 
and electronic brains going crazy.}

The browser shows it like this:

There is a young woman called Bracy, 
     whose clothes aren’t frilly or lacy. 
She expertly writes 
     about man-machine fights 
and electronic brains going crazy.

In a display scope, the word display must be immediately followed by a newline. This is because display works with lines: it forces you to start a new line for it to work with.

3.3.10. Indent.

The indent scope is like display, except that it maintains the alignment of the first nonblank character in each line with respect to the other lines. This is intended to support the indenting conventions used with programming languages.
      It’s easiest to show what indent does by means of an example. This is part of an Orson program that writes the value associated with an integer key k in a binary search tree r. (See any undergraduate Data Structures textbook for details.)

{indent 
(with 
  var bool g :− true 
  var tree s :− r 
 do (while g 
     do (if s = nil 
         then throw("keypunch error") 
         else (with int t :− k − s↑.key 
               do (if t < 0 
                   then s := s↑.left 
                   else if t > 0 
                        then s := s↑.right 
                        else write(''The value is: '') 
                             writeln(s↑.value) 
                             g := false)))))}

Note that the first character of the word with is aligned with the first character of the word do. The first characters of the words while and do are aligned in the same way, as are the first characters of the words if, then, and else. The result of the indent scope looks like this.

(with 
(wvar bool g :− true 
(wvar tree s :− r 
(do (while g 
(do (do (if s = nil 
(do (do (then throw("keypunch error") 
(do (do (else (with int t :− k − s↑.key 
(do (do (else (do (if t < 0 
(do (do (else (do (then s := s↑.left 
(do (do (else (do (else if t > 0 
(do (do (else (do (else then s := s↑.right 
(do (do (else (do (else else write(''The value is: '') 
(do (do (else (do (else else writeln(s↑.value) 
(do (do (else (do (else else g := false)))))

Even though the characters of the Roman font have different widths, the indent scope still maintains the alignments mentioned above. The first character of the word while is still aligned with the first character of the word do. The words with and do are still similarly aligned, and so are the words if, then, and else. This works even for text that uses multiple styles.

3.3.11. Orson.

The orson scope is like the indent scope, but it makes Orson source code look something like classical Algol 60 [NAU 1963]. In the example from the previous section, if you change the scope from indent to orson, you get this:

(with 
(wvar bool g :− true 
(wvar tree s :− r 
(do (while g 
(do (do (if s = nil 
(do (do (then throw("keypunch error") 
(do (do (else (with int t :− k − s↑.key 
(do (do (else (do (if t < 0 
(do (do (else (do (then s := s↑.left 
(do (do (else (do (else if t > 0 
(do (do (else (do (else then s := s↑.right 
(do (do (else (do (else else write(''The value is: '') 
(do (do (else (do (else else writeln(s↑.value) 
(do (do (else (do (else else g := false)))))

Operators appear without styles, plain names appear in italics, and reserved names appear in boldface. Character literals, comments, numeric literals, and string literals appear in their original styles. Quoted names also appear in their original styles, but their quote marks are italicized so they don’t look like apostrophes.
      The display, indent, and orson scopes produce horrible-looking HTML. In particular, the indent and orson scopes indent lines by writing text in white, the same color as the page background, which some browsers can’t print correctly on paper. These scopes are exceptions to the general rule that Bracy should produce HTML that’s easy to read.

3.3.12. Over.

The over scope displays mathematical proofs like those in type theory [CAR 1997], and is intended to help describe Orson’s type system. It has zero or more nested scopes. The odd-numbered scopes are shown separated by horizontal lines, and the even-numbered scopes label the lines. For example, this over scope:

{over 
{o{center {b type} [{i k}] {i int} ⊆ {b type} [] {i exe}} 
{o{left \ {i T}} 
{o{center [{i k}] {i int} ⊆ [] {i exe}} 
{o{left \ {i A}} 
{o{center {i int} ⊆ {i exe}} 
{o{left \ {i J}} 
{o{center {i true}}

appears as follows.

type [k] int ⊆ type [] exe

 T

[k] int ⊆ [] exe

 A

int ⊆ exe

 J

true

If an over scope has an odd number of nested scopes (as in the example), then the last scope is not followed by a horizontal line.

3.3.13. Narrow.

The narrow scope shows a series of scopes in a narrower column than would otherwise be used. (I could have said that it uses wider margins, but calling it widen would have been less clear.) This is useful to distinguish examples or long quotations from ordinary text; all the examples in this manual are shown using narrow. For example:

{narrow 
  {justify 
    After college, Thelma Bracy became a high school 
    mathematics teacher in Albany, New York. 
    A freak accident with an adding machine tragically 
    ended her life in October, 1967. 
    She never married or had children. 
    The year after her death, the Science Fiction 
    Association Press published an anthology of her work, 
    entitled {i Beyond Computation.} 
    It won her a special posthumous Hugo award in 1969.}} 
  {right 
    {i Thomas J. Chaswell}}}

This quotation is shown as follows:

After college, Thelma Bracy became a high school mathematics teacher in Albany, New York. A freak accident with an adding machine tragically ended her life in October, 1967. She never married or had children. The year after her death, the Science Fiction Association Press published an anthology of her work, entitled Beyond Computation. It won her a special posthumous Hugo award in 1969.

Thomas J. Chaswell

The narrow scope may be nested to obtain even narrower columns. However, most browsers can’t display text correctly if the columns become too narrow.

3.3.14. Comment.

A comment scope has an asterisk ‘*’ instead of a name, and it may contain zero or more characters. Bracy treats comment scopes as if they are blanks, so they may appear anywhere blanks may appear. However, they may not appear inside display, indent, or orson scopes, since these treat blanks as parts of lines.
      Since comment scopes are ignored, you can use them to add explanatory text to a Bracy source file. Here’s an example.

{* Copied right {C} 1919 by Thelma Bracy. 
   Last revision October 9, 1967. }

Any braces inside a comment scope must be matched: for each open brace, there must be a corresponding closing brace. If an unmatched brace appears, then it must be prefixed by a backslash. This allows nesting comment scopes, one inside the other. It also allows part of a Bracy source to be ‘‘commented out’’ by enclosing it in a comment scope.

3.3.15. Title.

The optional title scope defines the title at the top of a browser window. If it appears, then it must be the first scope in the Bracy source file (other than a comment scope). For example:

{title The Bracy Manual}

One or more words must appear inside the title scope, and any styles used by those words will be ignored.

3.3.16. Goto.

The goto scope sets up a link, either to a different web page (an external link), or to a different place in the current web page (an internal link). It must have a series of one or more words nested inside it.
      The first word says where the link points to. If the word has at least one character other than a Roman letter (A–Z, a–z) or a digit (0–9), then Bracy assumes it’s a URL, and sets up an external link. Bracy doesn’t test if the first word is really a URL. If it’s not, then what happens is up to your browser.
      If the URL is the only word in the goto scope, then you click on that word to follow the link. For example, this paragraph:

{justify 
  You should have received a copy of the {c Gnu} General 
  Public License along with Bracy. 
  If not, see {goto {t http://www.gnu.org/licenses/}}.}

looks like this.

You should have received a copy of the GNU General Public License along with Bracy. If not, see http://www.gnu.org/licenses/ .

Bracy displays links in boldface so they’ll stand out better from the surrounding text. Unvisited links are blue and visited links are purple, unless your browser decides otherwise.
      If there’s more than one word in the goto scope, then you click on any word other than the first, which is the URL. For example, this paragraph:

{justify 
  For a more realistic example, see the 
  {goto Manual.by source for the Bracy Manual.}}

looks like this:

For a more realistic example, see the source for the Bracy Manual.

A goto scope can appear anywhere that any other scope can. It can also appear inside the scopes center, justify, left, and right, which otherwise can have only words inside them. It can appear anywhere a word can appear.
      If the first word in a goto scope is made up only of letters and digits, then it’s considered to be an internal link. Internal links are defined using the label scope. The way label scopes work with goto scopes is described in the next section.

3.3.17. Label.

A label scope must have exactly one word nested inside it, which must be made up of Roman letters (A–Z, a–z) and digits (0–9). Any style information in the word is ignored, and the word itself is not displayed by your browser. For example, the first few lines of the Bracy Manual look something like this.

{title The Bracy Manual} 
{label top} 
{center 
  {b The {c Bracy} Manual}}

Now suppose you have a goto scope whose first word is top somewhere in your source file. If you click on the link established by that goto scope, then the browser will show you the part of the page that contains {label top}. For example, this paragraph:

{left Go to the {goto top} of the page.}

appears like this:

Go to the top of the page.

If you click on the word top, you’ll see the part of the Bracy Manual where the title appears. Similarly, this paragraph:

{left Go to the {goto top beginning of the manual.}}

appears like this:

Go to the beginning of the manual.

and if you click on one of the words beginning, of, the, or manual, the same thing will happen.
      Like the goto scope, the label scope can be nested inside the scopes center, justify, last, and right, which otherwise can have only words inside them. It can appear anywhere a word can appear.
      The goto and label scopes will work with any names, but you’ll get an error message if the same name appears in more than one label scope. However, Bracy doesn’t check if the name in a goto scope ever appears in a label scope, or vice-versa. If it doesn’t, then what happens is up to your browser.
      The program Bracx (pronounced bray-sex) is a cross reference generator for Bracy source files. It can help you keep track of goto and label scopes. For example, it can find goto scopes that have no matching label scopes, and label scopes that have no matching goto scopes. Bracx is not described here, but you can find out how to use it by typing the command man bracx.

3.3.18. Image.

The image scope must have exactly one word nested inside it. The word must have at least one character other than a letter or a digit, so that Bracy will treat it as a URL (like in a goto scope). Bracy assumes that the URL points to an image. It displays the image actual size, centered on the page, without borders. Bracy doesn’t test if the word in the image scope is really a URL, or if it really points to an image. If something is wrong with it, then what happens is up to your browser. For example, this pencil sketch of Thelma Bracy is displayed by the scope {image thelma.png}.

Curiously, no photographs of an adult Thelma Bracy are known to exist. She never had a driver’s license, or any form of photo ID. Even while employed as a teacher, she somehow avoided appearing in her high school’s yearbook. This sketch was drawn in the margin of a spiral notebook by one of her students, probably about 1965. Although crude, it is said to be a good likeness.

3.3.19. Insert.

The insert scope lets you use arbitrary HTML code. The scope must have exactly one word inside it, which is a pathname to a text file that contains the code. Bracy simply inserts the contents of this file at the point where the insert scope appears. For example, the scope {insert logigram.html} inserts the contents of the file logigram.html.
      If the file doesn’t exist, or if Bracy can’t read it, then you’ll get an error message. Bracy doesn’t test if the file really contains valid HTML. If it doesn’t, then what happens is up to your browser.
